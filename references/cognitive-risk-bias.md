# 인지 편향과 리스크 디브리프

## 6가지 편향과 리스크 평가에의 영향

### 1. 정상성 편향 (Normalcy Bias)

**정의**
- "위험한 일은 나에게 일어나지 않을 것"
- 미래가 과거처럼 계속되리라고 가정
- 통상적이 아닌 사건을 무시하거나 과소평가

**현실 데이터**

| 사건 | 정상성 편향 행동 |
|------|-----------------|
| 9/11 (2001) | 고층 건물 피난 명령 후 70%가 돌아옴 ("괜찮을 것" 생각) |
| Katrina (2005) | 수천 명이 피난 거부 ("나는 괜찮아") |
| 팬데믹 초기 (2020) | "동양 문제일 뿐, 우리는 안전" |
| 기업 위기 | "경쟁사는 망했지만 우리는 다르다" |

**리스크 스캔에서의 증상**
- "이건 우리에게 일어나지 않아"
- "지금까지 잘했으니 계속 잘 될 것"
- "극단적 시나리오는 현실성이 없어"

**대응**
- 기저율(base rate) 데이터 제시: "같은 산업의 경쟁사 중 80%가 이 위기 경험"
- 과거 실패 사례: "우리도 2008년에 당했던 일"
- Near-miss 추적: "지난번 운이 좋았던 것"

---

### 2. 가용성 휴리스틱 (Availability Heuristic)

**정의**
- 쉽게 떠오르는 것(기억하기 쉬운 것)의 확률을 과대평가
- 떠오르지 않는 것은 과소평가
- "내가 본 것 = 전체의 대표"라고 착각

**구체적 예**

**예 1: 데이터 보안**
- CEO가 "최근 페이스북 데이터 유출 뉴스" 봄
- → "사이버 리스크는 극히 높다" (극도로 과대평가)
- 실제: 회사는 중소 B2B, 타겟 가능성 낮음
- 결과: 과도한 보안 투자 (비용 대비 효과 낮음)

**예 2: 운영 리스크**
- "지난번 프로젝트에서 일정 지연 심했어" (3개월 지연, 매우 기억에 남음)
- → 모든 프로젝트를 이 기준으로 평가
- 실제: 그건 특수한 프로젝트였고 통상적으로는 정시 달성
- 결과: 모든 일정에 20% 버퍼 추가 (낭비)

**리스크 스캔에서의 증상**
- 최근의, 극적인 사건을 과대평가
- 오래되거나 "조용한" 리스크 무시
- 샘플 편향 (나의 경험만으로 확률 판정)

**대응**
- 체계적 리스크 리스트: 모든 축(R1~R6)을 순회해서 빠진 게 없나 체크
- 통계 제시: "우리 데이터로는 이 리스크의 확률은 2%, 당신은 20%로 봤는데 왜?"
- 익숙한 리스크 = 이미 관리되고 있을 수 있음

### 3. 낙관 편향 / 계획 오류 (Optimism Bias & Planning Fallacy)

**정의**
- 과제의 난이도, 기간, 비용을 체계적으로 과소평가
- "우리팀은 평균보다 좋으니까"
- "이번엔 잘 될 거야"

**실증 연구** (Flyvbjerg, 258개 프로젝트)

| 프로젝트 유형 | 평균 비용초과 | 평균 일정초과 |
|---|---|---|
| 철도 | +44.7% | +45% |
| 다리 | +33.8% | +27.5% |
| 도로 | +20.4% | +10% |
| IT 프로젝트 | +45~50% | +40~60% |

**흥미로운 결과**: "우리 회사가 특별하다"는 팀의 프로젝트도 분포는 동일.

**리스크 스캔에서의 증상**
- "우리는 4주 안에 개발하겠습니다" (기저율: 6주)
- "비용은 500만원" (기저율: 700만원)
- "리스크는 낮아. 우리팀 역량이 높으니까"

**대응 (Reference Class Forecasting)**

```
Step 1: Reference Class 선택
  "우리와 비슷한 프로젝트" 과거 사례 수집

Step 2: 분포 파악
  과거 프로젝트들의 실제 기간 데이터:
    낙관: 4주
    중위: 6주
    비관: 8주

Step 3: 우리의 위치
  "우리는 중위 팀" or "상위 팀"?
  → 중위라면 기저율은 6주

Step 4: 최종 추정
  6주 + 우리의 특수 요소 = 실제 추정
```

### 4. 서사 오류 (Narrative Fallacy)

**정의**
- 복잡한 현실을 너무 단순하고 일관성 있는 스토리로 재구성
- 그 스토리를 "진실"이라고 믿음
- 불확실성과 무작위성 무시

**예시**

**예 1: CEO 인사와 회사 실적**
- "새로운 CEO가 와서 매출 두 배로 증가"
- 스토리: CEO의 리더십 → 성공
- 현실: 시장 호황, 경쟁사 기술 부족, 시운의 중복 등
- 결과: 시장 침체 때 CEO 교체 (잘못된 인과)

**예 2: 투자 결정의 사후 합리화**
- "우리는 분석해서 이 회사가 성공할 것 알았어"
- 스토리: 데이터 기반 의사결정 = 성공
- 현실: 운 좋은 타이밍, 경쟁사 실수, 시장 상황 호황
- 결과: 같은 분석으로 다음 번에도 성공하리라 기대 (실패)

**리스크 스캔에서의 증상**
- "지난번에 우리가 이 리스크를 관리한 방법이 좋았어. 이번에도 그냥 그대로"
- "이 리스크는 실제로는 위협이 아니야. 언론이 과장한 것"
- 예측 불가능한 요소 제거

**대응**
- 미래의 불확실성 명시: "우리가 통제할 수 없는 것들"
- 대안 시나리오 개발: "만약 이게 반대로 되면?"
- 과거의 "행운"을 인정: "지난번 성공은 좋은 운도 있었다"

### 5. 확률 무시 (Neglect of Probability)

**정의**
- 확률 정보를 완전히 무시하고 임팩트만 봄
- 극도로 부정적이거나 긍정적인 서사가 등장하면, 확률은 무의미해짐

**예시**

**예 1: 보험 가입**
- "한국인이 암에 걸릴 확률: 30%대"
- "나는 절대 안 걸릴 것" (확률 무시)
- 하지만 "화재 사고" 뉴스 보면
- "나도 당할 수 있다" → 과잉 보험

**예 2: 비즈니스 리스크**
- "우리 시스템이 해킹될 확률: 5%"
- "그 확률은 무시하자. 대신 해킹되면 회사가 망한다는 건 확실하다"
- 임팩트가 극도로 크면 → 확률이 낮아도 보호 필수

**리스크 스캔에서의 증상**
- "이건 임팩트가 크니까 확률이 낮아도 신경 써야 해"
- "이건 확률이 높지만 임팩트가 작으니 무시해도 돼"
- P×I 매트릭스 활용 불충분

**대응**
- 양 축을 동시에 고려: 확률 낮지만 임팩트 높으면 → Yellow 이상
- 팻테일 리스크 명시: "확률은 1%지만, 이게 일어나면 회사 망한다"
- 선택: 이 극단적 리스크를 얼마나 보호할 것인가?

### 6. 유형성 휴리스틱 (Representativeness Heuristic)

**정의**
- "A가 B의 특징과 닮았으면, A는 B일 확률이 높다"
- 베이즈 정리 (기저율) 무시
- 스테레오타입에 의존

**예시**

**예 1: 회사 평가**
- "이 회사는 Google처럼 보인다 (젊은 직원, 혁신적 분위기)"
- → "이 회사도 Google처럼 성공할 거야"
- 실제: Google 같은 회사 99%는 실패
- 기저율: 신생 벤처 생존율은 10%

**예 2: 사람 평가**
- "이 사람은 은행원처럼 보인다 (보수적, 체계적)"
- → "금융 관련 일에 좋을 거야"
- 실제: 겉모습과 역량은 상관없음
- 기저율을 무시함

**리스크 스캔에서의 증상**
- "이 팀원은 유능해 보이니까 이 리스크는 낮을 거야"
- "이 기술은 최신 트렌드니까 성공할 확률이 높을 거야"
- "이런 유형의 프로젝트는 항상 성공했으니"

**대응** (베이즈 정리)

```
Base rate (기저율) 먼저:
  "같은 유형의 프로젝트 100개 중 성공 몇 개? 50개? 30개?"
  기저율 = 30/100 = 30%

우리의 특수성 고려:
  "우리 팀이 다른 팀보다 낫나?" → +10%
  "일정 여유가 더 있나?" → +5%
  
최종 확률:
  기저율 30% + 조정 15% = 45%
```

## 인지 편향 디브리프 프로토콜

### 실행 방식

리스크 스캔 **완료 후** (스캔 중이 아니라) 별도로 진행.

목표: "우리의 평가가 이 6가지 편향에 빠지지 않았나?"를 점검.

### 디브리프 테이블

| # | 편향 | 진단 질문 | 증상 | 대응 액션 |
|---|------|---------|------|---------|
| 1 | 정상성 편향 | "우리는 '이건 우리에겐 일어나지 않을 것'이라고 생각했나?" | "지금까지 잘했으니 계속 잘 될 것" | 기저율 데이터 제시. 과거 실패 사례 검토. Near-miss 추적 |
| 2 | 가용성 휴리스틱 | "최근에 본, 극적인 리스크는 과대평가하지 않았나?" | "페이스북 데이터 유출 뉴스 봤으니 우리도 위험" | 체계적 리스크 리스트로 균형 재점검. 통계 데이터 제시 |
| 3 | 낙관 편향 | "우리의 일정·비용 추정이 기저율보다 훨씬 낙관적인가?" | "우리는 4주에 끝내겠습니다" (기저율 6주) | Reference Class Forecasting 적용. 과거 데이터로 보정 |
| 4 | 서사 오류 | "복잡한 상황을 너무 단순한 스토리로 설명했나?" | "CEO 변경 = 성공" 같은 단순 인과관계 | 미래 불확실성 명시. 통제 불가 요소 나열. 시나리오 개발 |
| 5 | 확률 무시 | "임팩트가 크면 확률이 매우 낮은 리스크도 보호 필수라는 것을 빠뜨렸나?" | "확률 1%인데 보호 안 해" | 팻테일 리스크 명시적 검토. "만약 일어나면?" 시나리오 |
| 6 | 유형성 휴리스틱 | "스테레오타입에 맞으면 높은 확률이라고 생각했나?" | "젊은 팀이니까 혁신할 거야" | 베이즈 정리로 보정. 기저율 재확인. 정보 불균형 지적 |

### 실행 예시

**전체 리스크 스캔 후 디브리프 회의**

```
Q1: 정상성 편향
  팀원 A: "우리가 '이건 우리에겐 일어나지 않을 것'이라고 생각했나?"
  팀원 B: "네, 시장 침체 리스크는 낮다고 생각했어"
  A: "그럼 기저율은? 같은 산업 회사 중 몇 개가 지난 10년간 시장 침체 경험했나?"
  B: "아, 실제로 반 이상이 경험했네. 우리 확률을 40% → 60%로 올려야겠네"
  
Q2: 가용성 휴리스틱
  A: "최근 테슬라 자동주행 사고 뉴스 봤는데, 우리 자동화 리스크를 과대평가했나?"
  B: "음... 우리는 고객사 데이터를 다루지 않으니 임팩트는 다를 것 같아"
  A: "맞다. 뉴스는 극적이지만 우리의 특수성을 고려해야겠다. 확률은 유지, 임팩트는 하향"
  
Q3: 낙관 편향
  A: "개발 일정 추정이 4주라고 했는데, 과거 유사 프로젝트는?"
  B: "6주 평균이었어. 우리 팀이 더 빠르다고 생각했는데..."
  A: "Reference Class Forecasting으로 보정해. 4주 → 6주 기준으로 재평가"
  
Q4: 서사 오류
  A: "우리 CEO의 전략이 성공의 핵심이라고 생각했는데, 운의 역할도 있나?"
  B: "음, 지난번 시장 호황도 있었고..."
  A: "그럼 미래가 침체되면? 대안 시나리오 개발해서 전략 유연성 확인해야겠다"
  
Q5: 확률 무시
  A: "사이버 공격 확률은 2%인데, 임팩트가 극도로 크지 않나?"
  B: "네, 회사가 망할 수 있어"
  A: "2%라고 무시하지 말고, 임팩트 때문에 보호 수준을 올려야 한다"
  
Q6: 유형성 휴리스틱
  A: "신입 개발자는 최신 기술 배우기 빨라서 성공률이 높을 거라고 생각했나?"
  B: "네, 나이 어리고 똑똑한 사람이니까"
  A: "하지만 기저율은? 신입 개발자 프로젝트 성공률이 얼마였나?"
  B: "음, 50% 정도... 스테레오타입에 빠졌네"
```

### 디브리프 산출물

각 편향 검토 후:
- "이 편향이 우리의 평가에 영향을 주었는가?" → YES / NO / PARTIAL
- YES 또는 PARTIAL이면:
  - "어느 리스크에 영향을 주었는가?"
  - "확률/임팩트를 어떻게 수정할 것인가?"
  - "새로운 리스크를 추가해야 하는가?"

## 조직적 적용

### 문화적 전제

디브리프가 제 기능하려면:

1. **심리적 안전감** (Psychological Safety)
   - 편향을 지적해도 "바보"라고 여겨지지 않아야 함
   - "우리 모두 편향이 있다"는 조직 인식

2. **데이터 기반 문화**
   - "그것 같아" vs "기저율 데이터로는"
   - 수치적 검증을 자연스럽게

3. **정기적 실행**
   - 한 번으로 끝나지 말고, 프로젝트마다 실행
   - "편향 점검은 리스크 관리의 일부"라는 인식
